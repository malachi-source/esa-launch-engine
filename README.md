# ESA Launch Engine

The system that turns a client's onboarding form into a complete, AI-generated, human-approved
webinar launch (funnel, ads, follow-up) for Event Sales Agency.

## New here? Read this first

You don't need to know how to code. You direct Claude Code in plain English; it writes and runs
everything. Here's the whole workflow:

### One-time setup (each team member, once)
1. Install **git** (Mac: a popup installs the developer tools — click Install).
2. Make a free **GitHub account** at github.com and ask the repo owner to add you as a collaborator.
3. Get your own **Claude Code** access on your own computer.
4. Download ("clone") this repo — Claude Code will do this for you; just ask:
   *"clone the esa-launch-engine repo to my computer."*

### Daily work
1. Open Claude Code inside this project folder.
2. Tell it which workflow you own, e.g. *"I'm working on workflows/01-ad-scripts today."*
3. Work on the prompt with Claude. Test it. Improve it.
4. When you're done for the session, say:
   *"Update this folder's CLAUDE.md with what we changed and what's next, then save and push my work."*
   Claude handles all the git commands.

### The golden rules
- **Stay in your own workflow folder.** That's how we all work at the same time without collisions.
- **Always end a session by having Claude update the folder's `CLAUDE.md`.** That's our memory.
- **Never share GitHub logins.** Everyone uses their own account.

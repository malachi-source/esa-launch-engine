# 00-Foundation — working notes

## What this step does
The foundation step turns the raw client intake/marketing survey into three master reference docs: a Project Summary (who the client is and what they need from ESA), an Event Summary (what the event is and why it matters), and an Ultimate Audience Profile (deep copywriting-ready psychology of the target buyer). These three prompts run FIRST and their outputs feed every other workflow (ad copy, ad scripts, VSL, email/text nurture, sales script, lead forms, etc.).

## Inputs it needs (from the onboarding form / earlier steps)
Project Summary prompt:
- Client/Company name; event name, date(s), location; event type
- Website & registration link(s)
- Main objective(s) of the event
- Current situation (built/missing/challenges)
- Team and roles; CRM/platforms; marketing assets; budget/ad spend
- Previous event history; key notes, pains, requests

Event Summary prompt:
- Event name, date(s) & location, event type
- Website & registration link(s)
- Main topics / curriculum / speakers
- Transformation / outcomes
- Bonuses / special access; past event context

Audience Profile prompt:
- Uploaded survey responses, intake forms, and event/offer details

## Output it produces
- Project Summary: one-page client/event summary (overview, event details, objectives, current situation, marketing needs, tech needs, risks, ESA action plan)
- Event Summary: one-page event positioning doc (overview, what attendees learn, transformation/outcomes, why it matters, bonuses)
- Audience Profile (TAP): 5-7 page Fortune 500 quality document. MUST follow reference/tap-formatting-standards.md — no section breaks across pages, branded design with client colors, premium layout.
